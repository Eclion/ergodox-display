# Ergodox Display

A borderless, transparent, always-on-top desktop overlay (Linux & Windows)
that displays the layers of an Ergodox EZ keyboard layout — Symbols above
Base — and highlights the keys being pressed in real time.

Key events come straight from the keyboard over ZSA's Oryx raw-HID
live-training protocol (the same one Keymapp uses), so the overlay knows the
exact physical key and the active layer regardless of what the OS sees.

## How it works

- `layout.json` — the layout definition fetched from the Oryx GraphQL API
  (layout `yX3pz`, revision `XbQYA4`). `ui/layout.json` is the copy the
  frontend renders.
- `ui/geometry.json` — physical key positions and matrix coordinates for the
  Ergodox EZ, generated by `scripts/gen_geometry.py` from QMK's
  `LAYOUT_ergodox` definition (`assets/qmk/ergodox_ez_info.json`). Index `i`
  in this file corresponds to index `i` of each layer's `keys` array in the
  Oryx layout.
- The Rust backend (Tauri) finds the keyboard's raw HID interface
  (vendor `0x3297`, usage page `0xFF60`), sends `ORYX_CMD_PAIRING_INIT`, and
  streams keydown/keyup/layer events to the webview.

## Linux setup (one-time)

The raw HID device node is root-only by default. Install the udev rule
(you must be in the `plugdev` group, which is the Ubuntu default):

```bash
sudo cp assets/udev/50-zsa.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules && sudo udevadm trigger
```

Then unplug/replug the keyboard once.

## Build & run

Prerequisites: Rust (rustup), and on Linux the Tauri v2 system libraries
(`libwebkit2gtk-4.1-dev libxdo-dev libssl-dev libayatana-appindicator3-dev
librsvg2-dev`) plus `libudev-dev` (HID) and `libxkbcommon-dev
libxkbcommon-x11-dev` (OS keyboard-layout detection).

```bash
cd src-tauri
cargo run            # development
cargo tauri build    # installers (requires cargo install tauri-cli)
```

On Windows no special setup is needed; the WebView2 runtime is preinstalled
on Windows 10/11 and hidapi talks to the keyboard directly.

## Usage

- The overlay starts in **Movable** mode: drag it anywhere with the mouse.
- Use the tray icon → *Settings…* to switch to **Overlay** mode
  (click-through: mouse events pass through the window) and back.
- The active layer's board lights up as you switch layers; pressed keys are
  highlighted on the active layer's board.
- Key labels follow the OS keyboard language: printable keys show the
  character the active OS layout (e.g. AZERTY) actually produces, refreshed
  within ~2s of switching input language. Hover the layer name to see which
  OS layout is applied.
- Tray icon → *Quit* to exit. The window position and mode are remembered.

## Updating the layout

After changing the layout in Oryx, re-fetch it (replace hash/revision ids
with the ones from your Oryx URL) and copy it to the UI:

```bash
curl -s -X POST https://oryx.zsa.io/graphql -H 'Content-Type: application/json' \
  -d '{"operationName":"getLayout","variables":{"hashId":"yX3pz","revisionId":"XbQYA4","geometry":"ergodox-ez"},"query":"query getLayout($hashId: String!, $geometry: String, $revisionId: String!) { layout(hashId: $hashId, geometry: $geometry, revisionId: $revisionId) { title geometry revision { hashId title layers { hashId title position keys } } } }"}' \
  > layout.json
cp layout.json ui/layout.json
```
